# Passing Through — Настройка и Запуск

## 📋 Требования
- **Godot Engine 4.6.2** (или новее)
- Видеокарта с поддержкой OpenGL 3.3 / Vulkan (режим совместимости включён)

## 🚀 Быстрый старт

### 1. Открыть проект
- Запустите Godot 4.6.2
- Нажмите "Import" и выберите файл `project.godot` из папки `/workspace`

### 2. Проверка Autoload (Автозагрузка)
Проект уже настроен! В `Project Settings → Application → Autoload` должны быть:
- `GameState` → `res://scripts/global/game_state.gd`
- `EventManager` → `res://scripts/systems/event_manager.gd`
- `MemorySystem` → `res://scripts/systems/memory_system.gd`
- `GameManager` → `res://scripts/global/game_manager.gd`

Если их нет — добавьте вручную через кнопку **Add**.

### 3. Спрайты
Временные спрайты уже находятся в папке `assets/sprites/`:
- `player_placeholder.png` — персонаж
- `bg_far_placeholder.png` — дальний план
- `bg_mid_placeholder.png` — средний план
- `bg_near_placeholder.png` — ближний план

### 4. Запуск
- Откройте сцену `scenes/main.tscn`
- Нажмите ▶️ (F5) для запуска
- Нажмите **"НАЧАТЬ ИГРУ"** на главном экране

## 🎮 Управление

| Клавиша | Действие |
|---------|----------|
| **A / ←** | Движение влево |
| **D / →** | Движение вправо |
| **Пробел** | Взаимодействие / Выбор |
| **P** | Сделать фото (во время события) |

## 🔧 Основные системы

### GameState
Хранит:
- Треки (`action`, `observe`, `connect`)
- Архетипы (7 типов)
- Состояние (-2 до +2)
- Эхо-события
- Память (фото)

### GameManager
Управляет:
- Игровым циклом (утро → путь → сцена → фото → вечер → ночь)
- Сменой дней (1-15)
- Подсчётом событий за день
- Финальной интерпретацией

### EventManager
- Загружает события из JSON
- Фильтрует по дню, состоянию, трекам
- Управляет эхо-системой
- Запускает мини-игры

### MemorySystem
- Генерирует варианты фото
- Сохраняет выбранное фото дня
- Анализирует память для финала

## 🎯 Игровой цикл

Один день игры состоит из следующих фаз:

1. **🌅 УТРО** — начало дня, выбор направления
2. **🚶 ПУТЬ** — движение к событию (параллакс фон)
3. **🎭 СЦЕНА** — событие с выбором действий
4. **📸 ФОТО** — выбор кадра восприятия (2-3 варианта)
5. **🌙 ВЕЧЕР** — сохранение одного фото в память
6. **🌫 НОЧЬ** — рефлексия, возможные эхо-события

После 15 дней игра показывает **финальную интерпретацию** на основе:
- Доминирующего архетипа
- Статистики типов восприятия (participation/observation/connection)
- Выбранных фото и эмоций

## 📝 Как расширять

### Добавить новое событие
1. Создайте файл `data/events/my_event.json`
2. Скопируйте структуру из `starter_events.json`
3. Измените `id`, `text`, `choices`

Пример:
```json
{
  "id": "park_001",
  "title": "Парк",
  "text": "Ты в парке...",
  "day_min": 1,
  "day_max": 15,
  "choices": [...]
}
```

### Добавить новый архетип
Откройте `data/archetypes/archetypes.json` и добавьте:
```json
"new_archetype": {
  "name": "Название",
  "description": "Описание",
  "text_modifier": "modifier_key",
  "color_tint": [R, G, B, A],
  "final_interpretation": "Текст для финала"
}
```

### Изменить баланс
В `scripts/global/game_state.gd`:
- Начальные значения треков
- Диапазон состояния
- Формулы расчёта доминирующего архетипа

В `scripts/global/game_manager.gd`:
- `max_events_per_day` — количество событий в день (по умолчанию 3)

## ⚠️ Известные проблемы

### Ошибка видеокарты
Если видите `Your video card drivers seem not to support...`:
- Проект уже настроен на `renderer/rendering_method="compatibility"`
- Обновите драйверы NVIDIA
- В панели управления NVIDIA выберите Godot на дискретной видеокарте

### Ошибки скриптов
Все скрипты протестированы на Godot 4.6. Если возникают ошибки:
- Проверьте версию движка
- Убедитесь что Autoload подключены
- Перезагрузите проект

## 💡 Советы

- Все тексты в событиях поддерживают варианты для разных архетипов через `text_variants`
- Мини-игры запускаются автоматически если есть поле `minigame` в выборе
- Эхо-события возвращаются только если игрок пропустил оригинал (`skip_event: true`)
- Фото влияет на финальную интерпретацию через `perception_type`
- Игрок может менять линзу восприятия в `game_state.gd` (`current_lens`)
